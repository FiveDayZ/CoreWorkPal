# CoreCat Codex 分轮实施提示词包

本目录把 `CoreCat_后续几轮Codex实施提示词.md` 拆成 5 个独立 Markdown 文件，方便你按轮数依次喂给 Codex。

## 推荐执行顺序

1. `01_第1轮_VFX_Bus_核心高级状态VFX_DebugPanel.md`
2. `02_第2轮_补充交互状态_Runtime.md`
3. `03_第3轮_性能验收工具_QA面板_低功耗保护.md`
4. `04_第4轮_最终美术资产替换_资源检查器.md`
5. `05_第5轮_最终回归测试_验收清单_文档更新.md`

## 使用方法

每一轮都建议这样操作：

1. 复制对应 Markdown 文件全文给 Codex。
2. 等 Codex 完成修改。
3. 要求 Codex 输出：修改文件清单、测试结果、未完成项、下一轮建议。
4. 确认 `corepack pnpm typecheck`、`corepack pnpm test:corecat`、`corepack pnpm build` 通过。
5. Git commit 一次，再进入下一轮。

## 重要提醒

不要一次性把 5 轮全部喂给 Codex。否则它容易跳步骤、简化实现或把多个系统混在一起改。

# 附录 B：如果 Codex 试图接入大型动画库

如果 Codex 建议接入 Spine / Live2D / Rive / Pixi / Lottie，请回复：

```markdown
暂时不要接入 Spine / Live2D / Rive / Pixi / Lottie。

本项目当前阶段坚持轻量伪骨骼动画系统：

1. 分层 DOM/SVG/PNG 节点。
2. CSS transform / Web Animations API / requestAnimationFrame。
3. 局部 VFX，不做全屏 Canvas 重绘。
4. 常驻低功耗。
5. 后续如果最终美术和复杂形变确实需要，再单独评估专业骨骼工具。

请回到当前 Phase 的目标，继续按现有架构增量实现。
```

如果 Codex 说完成了，但实际只做了 transform pose，没有 VFX，请回复：

```markdown
这还不符合本轮目标。

本轮不是只做 pose，而是要完成对应 VFX 表现：

1. TemperatureCheck 必须有 CoolingWind / CoolingParticles / CoolingText。
2. MemoryCrowded 必须有 MemoryRamBox / MemorySteam。
3. Repairing 必须有 RepairSparks / HologramPanel。
4. DataSorting 必须有 DataCubes / pouch glow。
5. Celebrate 必须有完整 1800ms 一次性时间轴和 CelebrateBurst。
6. VFX 必须接入 VFX Bus，并且状态退出时可清理。
7. Debug Panel 必须能手动触发和验收。

请基于现有提交继续补齐，不要重构已经通过测试的基础运行时。
```
