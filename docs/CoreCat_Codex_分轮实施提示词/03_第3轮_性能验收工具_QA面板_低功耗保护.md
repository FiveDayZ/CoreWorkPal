```text

要求：

2. 监控 VFX 活跃数量。
3. 监控当前状态、上一状态、transition duration。
4. 监控是否处于 LowPower。
5. 监控是否暂停 VFX。
6. 监控最近 10 次状态切换记录。
7. 仅开发环境显示详细面板。
8. 生产环境不得出现调试 UI。

2. 不要每帧 setState 导致 React 重渲染。

# 二、低功耗保护策略

请完善 CoreCat LowPower 策略：

## 触发条件

以下条件应进入或建议进入低功耗：

1. 用户手动开启 Force LowPower。
2. Sleep / LowPower 状态。
3. 当前窗口不可见或失焦较久。
4. 系统高负载场景下，CoreCat 应减少 VFX 频率。
5. VFX 活跃数量超过阈值时自动降级。

## 降级策略

低功耗时：

1. 保留基础呼吸。
2. 保留 Sleep bubble。
3. 暂停高频粒子。
4. 降低 CoolingParticles / DataCubes 生成频率。
5. RepairSparks / CelebrateBurst 等一次性事件可以播放，但限制数量。
6. 不要完全冻结 CoreCat，除非用户开启极限低功耗。

# 三、VFX 安全阈值

请为 VFX Runtime 增加安全阈值：

```ts
export const CORECAT_VFX_LIMITS = {
  maxActiveParticles: 80,
  maxCoolingParticles: 24,
  maxDataCubes: 16,
  maxSteamPuffs: 8,
  maxSparkParticles: 24,
  maxCelebrateParticles: 48
}
```

要求：

1. 超过阈值时丢弃新粒子，而不是继续累积。
2. 状态退出必须清理对应 VFX。
3. Debug Panel / Performance Panel 显示当前粒子数量。
4. 增加测试验证不会无限累积。

# 四、动画 QA 验收面板

请在 Debug Panel 中增加 QA Tab 或新建 `CoreCatQaPanel.tsx`。

功能：

1. 一键播放全部状态验收序列。
2. 一键播放核心硬件状态序列：
   - Idle -> DataSorting -> MemoryCrowded -> TemperatureCheck -> Repairing -> Celebrate -> Idle
3. 一键播放交互状态序列：
   - BootWake -> Hover -> Click -> PanelOpen -> PanelClose -> Dragging -> DropLanding -> Idle
4. 显示每个状态是否完成自动回落。
5. 显示是否出现卡死状态。
6. 显示 VFX 清理是否成功。

要求：

1. QA 序列只在开发环境可用。
2. 不影响生产构建。
3. 不要把 QA 控制逻辑混进正式状态机主流程。

---

# 五、测试要求

请补充测试：

1. LowPower 会暂停高频 VFX。
2. VFX 粒子数量不会超过限制。
3. Sleep 状态下 CoolingParticles / DataCubes 不会继续生成。
4. QA 序列可以顺序触发状态。
5. 一次性状态不会卡住。
6. Performance Monitor 低频采样，不每帧触发 React UI 更新。

运行：

```powershell
corepack pnpm typecheck
corepack pnpm test:corecat
corepack pnpm build
```
