# 二、资产 Manifest

请新增资产 manifest：

```text
src/pet/corecat/assets/coreCatAssetManifest.ts
```

要求：

1. 定义每个骨骼节点对应资产路径。
2. 定义每个资产的 fallback。
3. 定义 pivot / anchor / default size。
4. 支持后续替换 PNG/SVG。
5. 不存在正式资产时继续使用 placeholder，不导致应用崩溃。

建议类型：

```ts
export type CoreCatAssetId =
  | 'shadow'
  | 'tail_base'
  | 'tail_mid'
  | 'tail_tip'
  | 'body_base'
  | 'arm_left'
  | 'arm_right_wrench'
  | 'arm_right_fan'
  | 'pouch'
  | 'head_base'
  | 'ears_left'
  | 'ears_right'
  | 'goggles'
  | 'eye_normal'
  | 'eye_blink'
  | 'eye_focused'
  | 'eye_dizzy'
  | 'eye_sleepy'
  | 'eye_glowing'
  | 'ram_box'
  | 'wrench_clone'
  | 'badge_star'
  | 'sleep_bubble'

export interface CoreCatAssetMeta {
  id: CoreCatAssetId
  path: string
  fallbackPath: string
  width: number
  height: number
  pivotX: number
  pivotY: number
  anchorNode: string
  required: boolean
}
```

# 三、资源检查器

请新增开发环境资源检查器：

```text
src/pet/corecat/assets/coreCatAssetValidator.ts
src/pet/corecat/debug/CoreCatAssetPanel.tsx
```

要求：

1. 检查必需资产是否存在。
2. 检查 manifest 中定义的资产是否能被 import 或加载。
3. 显示缺失资产列表。
4. 显示当前使用的是正式资产还是 placeholder。
5. 显示每个资产的 pivot / anchor。
6. 不要因为资产缺失导致应用崩溃。
7. 只在开发环境显示资源检查面板。

---

# 四、占位资产与正式资产解耦

要求：

1. 当前 DOM/SVG 占位形状可以继续作为 fallback。
2. 当 PNG/SVG 正式资产存在时，CoreCatLayer 应优先使用正式资产。
3. 每个骨骼节点仍必须是独立图层，禁止把最终美术合成成一整张图。
4. 资产替换不得破坏动画 transform。
5. 同一节点的 transform 应作用于外层 wrapper，而不是直接破坏图片 natural size。

# 五、素材 README

请生成 `src/pet/corecat/assets/README.md`，写清楚给美术或图像生成 AI 的交付规则：

1. 画布尺寸建议。
2. 每个部件命名。
3. 每个部件透明背景。
4. 不允许整张死图。
5. 部件边缘需要留安全透明区。
6. pivot 位置说明。
7. 导出 PNG / SVG 要求。
8. 高 DPI 缩放建议。
9. 正式替换流程。

# 六、测试要求

请补充测试：

1. manifest 包含所有 required asset。
2. 资产缺失时 fallback 不为空。
3. CoreCatLayer 可以渲染 placeholder。
4. 替换正式资产不会改变节点 ID。
5. 资源检查器不会出现在生产环境。

运行：

```powershell

corepack pnpm test:corecat
corepack pnpm build
```
