# CoreCat 骨骼动画动力学与 VFX 节点开发全集

## 1. 全状态骨骼动画参数总表 (Rigging & Physics Parameters)

以下参数直接对应 2D 骨骼的网格顶点变形（Mesh Deform）与旋转系数。

### 4.1 Idle (常驻异步静息)

- **设计意图**：提供永恒的生命力感，拒绝死气沉沉。
    
- **骨骼驱动核心**：
    
    - `body_base`：Y轴缩放运动。公式为 $scaleY = 1.0 + 0.015 \cdot \sin(\frac{2\pi \cdot t}{2800})$
        
    - `head_base`：随身体上下微移，但相位滞后 350ms，且伴随极轻微的 Z 轴摇摆（$\pm 0.5^\circ$）。
        
    - `tail`：应用 3 节点链式骨骼阻尼摆动。根骨骼静止，中骨骼旋转 $\pm 2^\circ$，尾端骨骼旋转 $\pm 4^\circ$，周期 3800ms。
        
- **情绪/眼神层**：`eye_normal.png`。每隔 5000ms~12000ms 随机触发一次 `eye_blink.png`（持续 100ms）。
    
- **特效(VFX)层**：无常驻特效。脚底阴影 `shadow.png` 透明度随呼吸在 `0.35` 到 `0.45` 之间呈正弦交替。
    

### 4.2 Hover (鼠标磁吸视线追踪)

- **设计意图**：让桌宠感知到用户的存在，产生“隔着屏幕看你”的交互感。
    
- **骨骼驱动核心**：
    
    - **磁吸目标锁定**：获取鼠标在画布内的相对坐标 $(X_{mouse}, Y_{mouse})$。
        
    - `head_base` & `goggles`：朝目标方向做平滑内插位移（Ease-Out）。最大偏移量 $\Delta X = \pm 5\text{px}$，$\Delta Y = \pm 3\text{px}$。
        
    - `goggles`（护目镜）由于处于视差前景，位移系数设为头部基座的 `1.2倍`，形成伪 3D 凸出立体感。
        
- **情绪/眼神层**：`eye_normal.png`。眼神位移系数设为头部基座的 `1.4倍`，牢牢锁定鼠标指针。
    
- **特效(VFX)层**：护目镜图层叠加一层高光流光滤镜（CSS Linear Shimmer），当鼠标进入的一瞬间闪过一道极细的冷青色光芒（持续 200ms）。
    

### 4.3 Click (鼠标点击受力回弹)

- **设计意图**：提供极佳的打击反馈与物理肉感（Squish and Stretch）。
    
- **骨骼驱动核心**：
    
    - **瞬间压缩（0-50ms）**：`body_base` 强行压扁。$scaleY = 0.92$, $scaleX = 1.04$, $translateY = 4\text{px}$。
        
    - **猛烈反弹（51-140ms）**：由于惯性拉伸冲高。$scaleY = 1.06$, $scaleX = 0.97$, $translateY = -3\text{px}$。
        
    - **阻尼谐振（141-300ms）**：应用阻尼正弦函数 $f(t) = e^{-\gamma t} \cdot \cos(\omega t)$ 进行 3 次快速衰减回弹，归于 $1.0$。
        
    - `arm_right_wrench`：在 51-200ms 期间，独立触发顺时针 360° 旋转骨骼动画。
        
- **情绪/眼神层**：点击瞬间切换为 `eye_focused.png`（单眼眨眼/wink 俏皮状态）。
    
- **特效(VFX)层**：小扳手旋转至最高点时，迸发 3-5 颗黄绿色微型像素星星，向右上方抛射并做自由落体淡出。
    

### 4.4 TemperatureCheck (硬件高温警报)

- **设计意图**：当 CPU/GPU 温度 > 75°C 时，表现出滑稽的工作危机感。
    
- **骨骼驱动核心**：
    
    - **高频惊慌微动**：整只猫骨骼在 X 轴向高频抖动（频率 15Hz，振幅 $\pm 0.8\text{px}$）。
        
    - `arm_right_wrench`：隐藏扳手，切换为“迷你科技折扇”或“气泵”图层。以 80ms 为周期，在 $-15^\circ$ 到 $+35^\circ$ 之间做极其暴力的极速往复挥动骨骼动画。
        
    - `head_base`：头往后缩，下巴微抬，表现出被热气热到的拟人神态。
        
- **情绪/眼神层**：`eye_focused.png` 切换到半眯的专注流汗眼神，额头叠加一层 `sweat_drop` 独立上下滑动的贴图层。
    
- **特效(VFX)层**：
    
    - **冷气风带**：从扇子挥舞方向生成 3 条半透明、带物理拖尾的青色风带（矢量弯曲）。
        
    - **粒子溢出**：猫咪脚底和身体周围持续向上漂浮蓝色冷光像素点（密度：12颗/秒，寿命 600ms，带轻微随机漂移）。
        
    - **护目镜字符**：护目镜反光层内亮起 10px 像素风的 “COOLING...” 红色字样跑马灯。
        

### 4.5 MemoryCrowded (内存吃紧重物压迫)

- **设计意图**：内存占用 > 85% 时，将抽象的物理内存满载具象化为重物托举。
    
- **骨骼驱动核心**：
    
    - **重力负荷承重姿态**：整个核心骨骼向下微蹲 $4\text{px}$。`body_base` 左右拉宽 $5\%$（表现被压扁感）。
        
    - `arm_left` & `arm_right_wrench`：双手均向上高高举起，骨骼关节呈紧绷受力折角，掌心处无缝钉死承接“巨大的RAM零件箱”。
        
    - **超负荷颤抖**：零件箱与双臂骨骼维持 `25Hz`、幅值为 `0.5px` 的垂直高频物理颤抖。两只小短腿（`legs`）每隔 3000ms 触发一次向外滑步滑开 2px 再惊慌收回的位移。
        
- **情绪/眼神层**：`eye_dizzy.png`（蚊香眼或极为吃力的交叉眼）。
    
- **特效(VFX)层**：托举的“RAM零件箱”外框高频交替呼吸闪烁橙红色的霓虹强光。每隔 800ms，箱子缝隙处由于挤压向外横向喷射出一股短暂的白色蒸汽蒸汽（像素烟雾粒子，持续 200ms 后消散）。
    

### 4.6 Repairing (工坊模块后台修复中)

- **设计意图**：用户执行清理、优化加速时，表现出极度可靠的硬核工作流状态。
    
- **骨骼驱动核心**：
    
    - **姿态扭转**：整体身体骨骼向左扭转 45°（侧身面对屏幕），腾出右侧空间。
        
    - `arm_right_wrench`：小扳手骨骼以 `160ms` 为周期，在 $+45^\circ$ 到 $-20^\circ$ 之间做有节奏的规律上下大力敲击动作。
        
    - `legs`：左脚或右脚伴随着敲击节奏，每 320ms 触发一次脚尖向下点地、脚跟抬起的“打拍子”骨骼动画。
        
- **情绪/眼神层**：`eye_focused.png`（坚毅、专注的工程师眼神）。
    
- **特效(VFX)层**：
    
    - 当小扳手下落击打到虚空受力点（即敲击帧）的瞬间，在受力中心点向外水平炸开 4 颗明黄色的火花粒子（Spark Particles），带初速度弹跳与重力下坠衰减。
        
    - 猫咪面前激活一块半透明的蓝色全息光幕（CSS Glassmorphism + 0.6 透明度），其上闪烁有伪代码字符流上下滚动的特效。
        

### 4.7 DataSorting (低负载日常扫描整理)

- **设计意图**：系统闲置时，表现猫咪在电脑里有条不紊、悠闲地做着家务。
    
- **骨骼驱动核心**：
    
    - `body_base`：维持常规的 Idle 柔和呼吸状态。
        
    - `arm_right_wrench`：抛弃扳手。右臂骨骼表现为极为流畅、舒缓的“虚空划拉”轨迹（贝塞尔曲线路径插值）。
        
- **情绪/眼神层**：`eye_normal.png`。眼神随着右手的划拉轨迹做平滑同步位移，显得眼神灵动。
    
- **特效(VFX)层**：
    
    - 在猫咪身体右侧 `20px` 处，利用 Canvas 动态生成一个个规律浮现的 `8x8像素` 淡蓝色和淡绿色数据方块。
        
    - 当猫咪的手臂划过方块时，方块触发一个向内收缩、吸入并淡化消失的动画，同时在随身工具包（`pouch`）节点处亮起一瞬间的青色微弱外发光。
        

### 4.8 Sleep / LowPower (系统休眠/低功耗挂机)

- **设计意图**：当系统进入不打扰模式或挂机时，提供极具安全感的治愈陪伴。
    
- **骨骼驱动核心**：
    
    - **缩团坐姿**：猫咪骨骼由站姿无缝切换为坐姿/蜷缩姿态。各个分层骨骼节点（头、手、尾）向身体中心点靠拢拉近，可视高度自然降低至 `96px` 左右。
        
    - `goggles`：护目镜骨骼向上滑动 $8\text{px}$，乖巧地推在额头上方。
        
    - **极慢深度呼吸**：整体 scaleY 的拉伸周期延长至极其舒缓的 `5200ms`，呼吸幅度略微加大到 `2%`。
        
    - `tail`：尾巴不再做波浪摆动，而是轻柔地环抱盘旋在身体基座前方，随身体呼吸做同频共振。
        
- **情绪/眼神层**：`eye_sleepy.png` 彻底闭合，转化为两条安详的像素弧线。
    
- **特效(VFX)层**：
    
    - 彻底关闭脚底阴影的变化和其他一切高频特效。
        
    - 鼻尖节点处挂载一个 12px 宽的半透明浅蓝色“呼噜泡泡（`sleep_bubble`）”。泡泡的缩放与深度呼吸曲线完全同步：身体吸气时泡泡缩小至 `0.1`，身体呼气时泡泡膨胀至 `1.0`（带 1.5px 极弱的蓝色高光圈），并在达到最大值时触发一次随机闪烁。
        

## 2. 动效场景衔接控制层规范 (State Transition Matrix)

为了防止两套完全不同的骨骼动作发生生硬的画面截断，Codex 前端引擎层必须通过以下过渡帧权重融合（Cross-fading）进行动画切流：

Plaintext

```
               ┌───────────────┐
               │     IDLE      │◀──────────────────────────────┐
               └───────────────┘                               │
                   │       ▲                                   │
     (Hover 进入)  │       │ (Hover 移出)                      │
                   ▼       │                                   │
               ┌───────────────┐                               │
               │   TRACKING    │                               │
               └───────────────┘                               │
                   │       ▲                                   │
       (点击触发)  │       │ (3次阻尼回弹完毕)                 │
                   ▼       │                                   │
               ┌───────────────┐                               │
               │CLICK RESPONSE │                               │
               └───────────────┘                               │
                       │                                       │
            ┌──────────┴──────────┬──────────┐                 │
            ▼                     ▼          ▼                 │
     (温度 >75°C)          (内存 >85%)   (模块升级)            │(庆典/修复结束)
    ┌───────────────┐    ┌───────────┐  ┌───────────┐          │
    │  TEMP CHECK   │    │MEM CROWDED│  │ CELEBRATE │──────────┘
    └───────────────┘    └───────────┘  └───────────┘
```

### 权重混合规则 (Blending Rules)

1. 当由 `Idle` 切换至 `MemoryCrowded` 或 `TemperatureCheck` 时，代码层严禁直接重置当前帧。必须读取当前部件的位移矩阵参数，在 `160ms` （过渡窗口）内通过线性插值（LERP）平滑过渡到新动作的骨骼初始帧。
    
2. `Celebrate` 属于一次性高潮表现力动作（动作总长 1800ms）。动作播放完毕后，右臂扳手与脚底粒子特效自然淡出，身体主骨骼在 `240ms` 内平滑松弛（Ease-In-Out）并重新接入 `Idle` 的呼吸正弦周期。
